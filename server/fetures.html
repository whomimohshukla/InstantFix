Fixlyo – Features, Roles & Product Plan

High-level product summary based on current backend (Prisma + Node) and demo UIs for customer, technician, and admin. Use this as a briefing for design & frontend teams.

1. Roles & responsibilities

1.1 Customer (Web / App)

  • Browse categories and services with images, pricing, ratings.
  • Set location / pincode and manage multiple addresses.
  • Create bookings with preferred time slot and notes.
  • Pay online (Razorpay) or choose postpaid cash/UPI to technician.
  • Track booking status (PENDING → CONFIRMED → ASSIGNED → ENROUTE → IN_PROGRESS → COMPLETED → CANCELED).
  • Rate and review completed bookings.
  • Raise disputes and view dispute resolution status.
  • View notifications, profile, rewards/loyalty, saved services.

1.2 Technician (Web dashboard / App)

  • Sign up / login as a verified Fixlyo technician.
  • View todays jobs, job details, and navigation info.
  • Update booking status (ENROUTE, IN_PROGRESS, COMPLETED) via technician APIs.
  • See payment mode & amount to collect; confirm when cash/UPI is collected.
  • View earnings created when bookings complete.
  • View payout history generated by admin.
  • Manage availability and service areas (location management).
  • Manage portfolio images (before/after work) using generic Media model.
  • Maintain profile and track verification/KYC status.

1.3 Admin (Internal panel)

  • Monitor overall bookings, revenue, and live operations.
  • View and manage bookings, update statuses, reassign technicians, and trigger dispatch/auto-matching.
  • Manage disputes end-to-end (list, view, resolve with or without refund).
  • Monitor payments (Razorpay orders/payments/refunds) and reconcile with bookings.
  • Generate and manage technician payouts from unpaid earnings.
  • Manage catalog (categories, services, media/images).
  • Review, verify, and manage technicians (KYC, activation, suspension).
  • View and manage user roles (CUSTOMER, TECHNICIAN, ADMIN).
  • View admin audit logs for traceability.

2. Current backend capabilities (high level)

  • Bookings Full lifecycle with strict status transitions and technician flows.
  • Payments Razorpay integration for orders/webhooks, payment status stored per booking.
  • Postpaid cash Support for cash/UPI to technician with earnings & payouts.
  • Disputes Customer dispute creation + admin list/get/resolve with optional refund.
  • Media Generic Media model to attach images to services, technicians, bookings.
  • Payouts & earnings Earning per completed booking + admin payout batches.
  • Audit logs Admin actions logged for transparency.
  • Geo Basic geo/service-area structure for future auto-matching.

4. Domains / apps structure

  • Main customer app – e.g. fixlyo.com
    • Marketing site + logged-in customer experience.
  • Technician dashboard – e.g. tech.fixlyo.com
    • Job management, payments/earnings, availability, portfolio.
  • Admin panel – e.g. admin.fixlyo.com
    • Ops, support, finance, catalog and risk management.

All three can share a monorepo with shared UI and API client packages, but remain clearly separated in routing and branding.

5. Future goals & differentiation vs Urban Company / others

5.1 Short-term goals

  • Finalize all backend flows (dispatch/auto-matching, payouts, notifications) and harden error handling.
  • Implement polished frontends for all three roles using a consistent design system.
  • Add real-time status & live tracking for technicians (Socket.IO + map UI).
  • Improve discovery: better search, filters, and recommendations.

5.2 Medium-term differentiation ideas

  • Technician-first experience – clear earnings, payouts, and growth path so technicians prefer Fixlyo over others.
  • Predictable pricing – transparent price estimator per service, dynamic pricing by area/time, clear breakup for customer.
  • Quality guarantee – strong rework policy, faster dispute resolution with partial refunds, and visible trust badges.
  • Personalization – show relevant services, repeat bookings, and loyalty offers based on history and location.
  • Lightweight B2B – in future, allow offices/PGs to book recurring maintenance plans.

5.3 Gaps to fill vs mature platforms

  • Full auto-matching / dispatch engine using location, skills, and real-time capacity.
  • Deep review & rating system (photos, tags like professional, on-time, etc.).
  • Promotions & coupons system (campaigns, referral codes, cashback).
  • More advanced risk systems (fraud signals, abuse detection).
  • Full analytics dashboards for operations and finance.

6. Simple logo concept

For a first, simple direction that a designer can professionalize:

Name: Fixlyo

  • Mark: A minimal wrench + lightning bolt icon (suggesting instant fix) inside a rounded square.
  • Typography: Clean geometric sans (e.g. similar vibe to SF Pro / Inter) with slightly rounded corners.
  • Letter play: Emphasize the xo at the end with a subtle color accent to make the wordmark unique.
  • Color direction (can be refined later): primary cool blue or teal for trust + a warm accent (orange) for instant.

This keeps the logo simple enough for app icons, favicons, and technician T-shirts, while still feeling modern.
